## Завдання 9.1

## Завдання 9.3
Напишіть програму, яка від імені root копіює файл, який вона перед цим створила від імені звичайного користувача. Потім вона повинна помістити копію у домашній каталог звичайного користувача.
Далі, використовуючи звичайний обліковий запис, програма намагається змінити файл і зберегти зміни. Що відбудеться?
Після цього програма намагається видалити цей файл за допомогою команди rm. Що відбудеться?
### Відповідь
* Спроба зміни файлу: Оскільки копія була створена від імені root, вона за замовчуванням належить суперкористувачу. Якщо звичайний користувач спробує змінити її вміст, система видасть помилку "Permission denied", оскільки у користувача немає прав на запис у цей файл;
* Спроба видалення файлу: Це спрацює успішно. У Linux право на видалення файлу визначається правами доступу до директорії, у якій цей файл розташований, а не правами на сам файл. Оскільки домашній каталог належить користувачу, він має право видаляти будь-які об'єкти всередині нього, навіть якщо вони належать root.
### Програмна реалізація
Для виконання завдання було розроблено програму, яка автоматизує створення файлу та його копіювання з підвищеними привілеями через sudo, що робить root власником нової копії. Далі програма послідовно викликає стандартні команди оболонки для перевірки можливості запису та видалення.
### Результат виконання
```
gcc -Wall task_9_3.c -o task_9_3
```
**Вивід терміналу:**
```

Copied to 'root_copy.txt' as root (UID 0).
Attempting to modify 'root_copy.txt' as current user...
sh: 1: cannot create root_copy.txt: Permission denied
   Result: Modification FAILED (Permission denied).
Attempting to delete 'root_copy.txt' using rm...

   Result: Deletion SUCCESSFUL (Directory permissions allow this).
```
