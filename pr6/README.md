# Практична робота 6
## Завдання 4
Дослідити механізм copy-on-write, створивши програму з fork() та вимірявши кількість сторінкових переривань під час модифікації пам’яті за допомогою perf.

### Програмна реалізація
Для виконання завдання було розроблено програму, яка показує як Linux економить пам'ять під час створення дочірніх процесів за допомогою fork().
* Ми виділяємо великий блок пам'яті (200 МБ) та ініціалізуємо його, щоб гарантувати виділення фізичних сторінок;
* Після fork() батьківський та дочірній процеси спільно використовують одну й ту саму область пам'яті;
* Коли дочірній процес починає змінювати пам'ять, ядро Linux автоматично створює окремі копії змінених сторінок лише для нього.

### Результат виконання
Для вимірювання кількості сторінкових переривань використовується утиліта perf. Оскільки за замовчуванням доступ до цих даних обмежений безпековими налаштуваннями ядра, перед запуском необхідно послабити параметри perf_event_paranoid:
```
# Послаблюємо налаштування безпеки ядра для доступу до perf
sudo sysctl -w kernel.perf_event_paranoid=1

# Компіляція
gcc -Wall task_4.c -o task_4

# Вимірювання переривань
perf stat -e page-faults ./task_4
```
**Вивід програми**
```
perf stat -e page-faults ./task_4
Memory allocated and initialized. Parent PID: 6523
Child process writing to memory...
Child process finished writing.
Parent process: child finished.

 Performance counter stats for './task_4':

            76,901      page-faults                                                           

       4.407361668 seconds time elapsed

       0.155568000 seconds user
       4.223752000 seconds sys

```
Велика кількість page-faults під час роботи дочірнього процесу доводить, що Linux не копіює всю пам'ять одразу, а робить це лише тоді коли дані реально змінюються. Великий показник часу sys (4.22 с) підтверджує, що ядро системи активно працювало обробляючи ці переривання та виділяючи нові сторінки пам'яті для дочірнього процесу.
