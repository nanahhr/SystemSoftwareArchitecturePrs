## Завдання 8.1
Чи може виклик count = write(fd, buffer, nbytes); повернути в змінній count значення, відмінне від nbytes? Якщо так, то чому? Наведіть робочий приклад програми, яка демонструє вашу відповідь.
### Відповідь
Так, може. Це називається partial write. Основні причини:
* Недостатньо місця на диску;
* Системний виклик був перерваний сигналом до того, як усі дані були записані;
* Досягнуто максимальний дозволений розмір файлу для процесу.
### Програмна реалізація
Для виконання завдання було розроблено програму, яка ініціює операцію запису у файл за допомогою системного дескриптора. Програма порівнює кількість переданих для запису байтів із значенням, яке повернула функція write().У наведеному прикладі програма демонструє структуру перевірки на випадок виникнення partial write.
### Результат виконання
```
gcc -Wall task_8_1.c -o task_8_1
```
**Вивід терміналу:**
```
./task_8_1
Requested bytes: 12
Actual written bytes: 12
Status: Full write successful.
```

## Завдання 8.2
Є файл, дескриптор якого - fd. Файл містить таку послідовність байтів: 4, 5, 2, 2, 3, 3, 7, 9, 1, 5. У програмі виконується наступна послідовність системних викликів:
* lseek(fd, 3, SEEK_SET);
* read(fd, &buffer, 4);
де виклик lseek переміщує покажчик на третій байт файлу. Що буде містити буфер після завершення виклику read? Наведіть робочий приклад програми, яка демонструє вашу відповідь.
### Відповідь
Індексація у файлі починається з 0. Початковий стан: [0:4, 1:5, 2:2, 3:2, 4:3, 5:3, 6:7, 7:9, 8:1, 9:5]. lseek(fd, 3, SEEK_SET) переміщує покажчик на зміщення 3 (це четвертий байт за рахунком), де знаходиться значення 2. read(fd, &buffer, 4) зчитує 4 байти, починаючи з цієї позиції (байти з індексами 3, 4, 5, 6).
Після завершення викликів буфер буде містити послідовність: 2, 3, 3, 7.
### Програмна реалізація
Для виконання завдання було розроблено програму, яка імітує роботу з двійковим файлом, що містить задану послідовність чисел. Програма спочатку записує масив байтів у тимчасовий файл, після чого використовує системний виклик lseek() для позиціювання на третє зміщення від початку файлу. Далі за допомогою виклику read() проводиться зчитування 4 байтів у буфер.
### Результат виконання
```
gcc -Wall task_8_2.c -o task_8_2
./task_8_2
```
**Вивід терміналу:**
```
./task_8_2

Bytes read: 4
Buffer content: 2 3 3 7 
```

## Завдання 8.4
Виконайте наступну програму на мові програмування С:
int main() {
  int pid;
  pid = fork();
  printf("%d\n", pid);
}
Завершіть цю програму. Припускаючи, що виклик fork() був успішним, яким може бути результат виконання цієї програми?
### Відповідь
Результатом виконання цієї програми буде виведення двох чисел, кожне з нового рядка. Припускаючи, що fork() успішний: 
* Батьківський процес отримає в змінну pid значення PID новоствореного дочірнього процесу;
* Дочірній процес отримає в змінну pid значення 0. 
### Програмна реалізація
Для виконання завдання було реалізовано програму, яка викликає функцію fork() та виводить отримане значення змінної pid. У коді додано перевірку на помилку створення процесу (значення -1). Оскільки планувальник завдань Linux керує порядком виконання процесів самостійно, порядок виведення чисел (спочатку 0 чи спочатку PID) може змінюватися при різних запусках програми.
### Результат виконання
```
gcc -Wall task_8_4.c -o task_8_4
./task_8_4
```
**Вивід терміналу:**
```
./task_8_4
3695
0
```
